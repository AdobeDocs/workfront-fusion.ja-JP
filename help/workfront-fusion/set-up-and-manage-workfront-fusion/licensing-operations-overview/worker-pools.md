# 作業者プール

ワーカープールは、特定の組織専用のWorkfront Fusion 処理リソースの量です。 Fusion の操作と処理はすべて、組織に割り当てられたワーカープールのコンテキスト内で行われます。

ワーカープールを使用すると、シナリオをより効率的に実行でき、Fusion の処理能力について他の組織と競合する可能性がなくなります。

## ワーカープールの概要

ワーカープールを使用すると、シナリオの実行を並行して処理できます。 各ワーカープールは、次の項目に関連付けられます。

* 労働者の数。
* 実行キュー。

シナリオが実行されると、ワーカープールの実行キューに渡されます。 プール内のワーカーは、継続的に実行プールからタスクを取り込み、並行して処理します。

実行キューの深度が深くなり、既存のすべてのワーカーが使用されている場合、Workfront Fusion は、ワーカーを追加することでプールを自動的にスケーリングします。 このスケーリングは動的で、イベント主導型です。つまり、お客様や組織が何もしなくても、容量はリアルタイムの負荷に基づいてエクストラクトまたはコントラクトされます。

Workfront Fusion チームは、プールの使用率とスケーリング動作を継続的に監視し、しきい値やパターンを確認して、使用の増加に合わせて一貫したパフォーマンスを確保できます。
